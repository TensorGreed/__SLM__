# BrewSLM Docs

BrewSLM helps you build, evaluate, optimize, and export **domain-specific small language models (SLMs)** without stitching together five separate tools.

This documentation is written for a **new or early-career ML engineer** who wants to go from idea to a working model with minimal friction.

## What You Can Do Here

- Create a project with guided defaults and starter packs.
- Ingest and normalize domain data.
- Train and fine-tune small models with safety checks.
- Evaluate quality with gate policies and remediation guidance.
- Export artifacts for server, edge, browser, and mobile runtimes.

## Recommended Learning Path

1. Start with [Quickstart](getting-started/quickstart.md).
2. Follow [Build Your First Project](getting-started/first-project.md).
3. Learn the full [Pipeline Overview](workflows/pipeline-overview.md).
4. Deep dive into each stage only when needed.

## Core Principles

- **Measured over guessed**: whenever possible, runtime metrics should come from real measurement.
- **Safe fallback**: when a measured path is unavailable, BrewSLM can fallback with clear provenance.
- **Actionable blockers**: if you are blocked, the platform should tell you exactly what to fix next.
- **Novice-first UX**: sensible defaults before advanced tuning.

## If You Are Short on Time

Use this minimum path:

1. Create project.
2. Upload small clean dataset.
3. Run wizard/autopilot with defaults.
4. Run evaluation and inspect failures.
5. Export to one target and run smoke test.

You can get this done in under an hour for a first pass.
